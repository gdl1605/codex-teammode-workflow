# Examples

No concrete sample project is shipped in v0.1.0.

Future examples should be:

- generic enough to apply across product domains
- fully sanitized
- free of product names, customer names, screenshots, domains, credentials, database identifiers, or private architecture details
- clear about which files were copied verbatim and which files were adapted for the target repo

Useful future examples:

- empty target repo install
- existing `AGENTS.md` + new `CLAUDE.md` merge
- existing `docs/` merge
- Codex-to-Claude Code handoff behavior
